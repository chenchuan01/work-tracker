# 配置管理说明

## 功能概述

系统已经实现了将模型参数和提示词保存到数据库的功能。所有配置数据都存储在 SQLite 数据库的 `config` 表中。

## 数据库表结构

### config 表

```sql
CREATE TABLE IF NOT EXISTS config (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updatedAt INTEGER NOT NULL
)
```

**字段说明：**
- `key`: 配置键名（主键）
- `value`: 配置值（JSON 字符串格式）
- `updatedAt`: 更新时间戳（毫秒）

## 已保存的配置项

### 1. 模型配置 (model_config)

**配置键：** `model_config`

**配置内容：**
```json
{
  "apiKey": "sk-...",           // OpenAI API Key
  "baseURL": "https://api.openai.com/v1",  // API 基础地址
  "modelName": "gpt-3.5-turbo", // 模型名称
  "systemPrompt": "你是一位专业的工作助理..."  // 系统提示词
}
```

**使用场景：**
- AI 工作总结生成
- 每日金句生成
- 新闻总结

### 2. 每日金句缓存 (daily_quote_YYYY-MM-DD)

**配置键：** `daily_quote_2026-03-05`（日期动态变化）

**配置内容：** 字符串格式的每日金句

**用途：** 缓存每天生成的金句，避免重复调用 API

## API 接口

### 获取配置

```bash
GET /api/config/:key
```

**响应示例：**
```json
{
  "value": {
    "apiKey": "sk-...",
    "baseURL": "https://api.openai.com/v1",
    "modelName": "gpt-4",
    "systemPrompt": "你是一位专业的工作助理"
  }
}
```

### 保存配置

```bash
POST /api/config/:key
Content-Type: application/json

{
  "value": {
    "apiKey": "sk-...",
    "baseURL": "https://api.openai.com/v1",
    "modelName": "gpt-4",
    "systemPrompt": "你是一位专业的工作助理"
  }
}
```

**响应示例：**
```json
{
  "success": true
}
```

## 前端使用示例

### 加载配置

```typescript
import { apiClient } from './api/client';
import { ModelConfig } from './utils/prompt';

// 加载模型配置
const loadConfig = async () => {
  const saved = await apiClient.getConfig<ModelConfig>('model_config');
  if (saved) {
    setModelConfig(saved);
  }
};
```

### 保存配置

```typescript
// 保存模型配置
const handleSaveApiKey = async () => {
  await apiClient.setConfig('model_config', modelConfig);
  alert('配置已保存');
};
```

3. **查看配置数据：**
   ```sql
   SELECT
     key,
     value,
     datetime(updatedAt/1000, 'unixepoch', 'localtime') as updated_time
   FROM config;
   ```

4. **查看模型配置：**
   ```sql
   SELECT * FROM config WHERE key = 'model_config';
   ```

## 配置界面

在前端界面中，点击右上角的"设置"按钮（⚙️图标），可以配置：

1. **API Key**: OpenAI 或兼容 API 的密钥
2. **Base URL**: API 基础地址（支持自定义端点）
3. **模型名称**: 如 gpt-3.5-turbo, gpt-4, claude-3 等
4. **系统提示词**: 自定义 AI 助手的角色和总结要求

所有配置保存后会立即写入数据库，并在下次启动时自动加载。

## 数据安全

- API Key 等敏感信息存储在服务器端数据库中
- 数据库文件位于 Docker 数据卷，与容器隔离
- 建议定期备份数据库文件
- 生产环境建议对数据库文件进行加密

## 备份与恢复

### 备份配置
```bash
# 备份整个数据库
cp ~/docker/data/work-tracker/work-tracker.db ~/backup/work-tracker-$(date +%Y%m%d).db

# 仅导出配置表
sqlite3 ~/docker/data/work-tracker/work-tracker.db ".dump config" > config-backup.sql
```

### 恢复配置
```bash
# 恢复整个数据库
cp ~/backup/work-tracker-20260305.db ~/docker/data/work-tracker/work-tracker.db

# 导入配置表
sqlite3 ~/docker/data/work-tracker/work-tracker.db < config-backup.sql
```
